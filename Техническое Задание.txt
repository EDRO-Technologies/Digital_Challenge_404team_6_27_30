Техническое Задание: Разработка платформы адаптации
1. Цель работ
Создать программный комплекс, позволяющий автоматизировать процесс ввода в должность новых сотрудников ООО «Газпром трансгаз Сургут» с использованием технологий искусственного интеллекта и геймификации.
2. Требования к системе
2.1. Требования к интерфейсу (Frontend)
1. Адаптивность: Корректное отображение на мобильных устройствах (PWA) и десктопах.
2. Дизайн: Строгое соответствие брендбуку (цвета, шрифты), при этом использование современных UI-паттернов (карточки, анимации).
3. Аватар: Наличие визуального образа наставника (фото/видео), сопровождающего пользователя на всех экранах.
2.2. Требования к функционалу Бэкенда
1. API: RESTful API на базе FastAPI. Документация в формате OpenAPI (Swagger).
2. Безопасность:
   * Хранение паролей в хешированном виде (bcrypt).
   * Доступ к методам API только по Bearer Token (JWT).
   * Разделение прав доступа (RBAC) на уровне эндпоинтов.
3. База данных: Реляционная СУБД PostgreSQL. Использование ORM (SQLAlchemy) для работы с данными. Миграции через Alembic.
2.3. Требования к ИИ-модулю (RAG)
1. Локальность: Все компоненты (LLM, Vector DB) должны работать внутри контура (в Docker-контейнерах), без обращения к внешним API (OpenAI и т.д.) для безопасности данных.
2. Контекст: Поиск ответов должен осуществляться только по документам, загруженным в соответствующий трек адаптации.
3. Скорость: Ответ на вопрос не должен превышать 10-15 секунд (при использовании GPU).
3. Этапы сдачи работ (Roadmap)
1. Этап 1: Прототип ядра (Завершено).
   * Развернута инфраструктура (Docker, Postgres, Chroma, Ollama).
   * Реализована базовая схема БД.
2. Этап 2: Бизнес-логика (В процессе).
   * Реализация API для квестов и тестов.
3. Этап 3: Фронтенд MVP.
   * Верстка экранов сотрудника.
   * Интеграция с API.
4. Этап 4: Административная панель.
   * Интерфейсы для HR и Наставников.
5. Этап 5: Релиз.
   * Финальное тестирование и развертывание.
